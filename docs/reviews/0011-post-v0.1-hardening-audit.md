## 3. CI hardening

Trigger surface on `main` (`.github/workflows/ci.yml`, last
modified in commit 8f73720 + 58):

```yaml
on:
  push:
    branches: [main]
    tags: ['v*']            # tag-ref push fires CI
  pull_request:
    branches: [main]
  workflow_dispatch: {}     # UI re-run button

permissions:
  contents: read            # explicit, lowest privilege

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true
```

Forbidden-imports CI pattern verified:

```
! grep -RE "^\s*(import|from)\s+(ccxt|openai|anthropic|langchain|
                                  web3|binance|btcturk|pybit|okx|
                                  gate_api|kucoin|huobi|bitfinex|
                                  kraken)" sentinel/ tests/
```

Reproduced locally: **exit 1 (no match) — clean.**

The three matches found in `sentinel/constitution/invariants.py`
lines 317-327 are inside **string literals describing what is
forbidden** (the invariant catalog narration); the CI grep
anchors on `^\s*(import|from)\s+` so those strings cannot
trigger it. This is correct behavior — the catalog must be
able to NAME the forbidden SDKs in its own description without
itself becoming a false-positive.

Forbidden-outputs CI pattern verified:

```
! grep -RE "\"(BUY|SELL|EXECUTE_REAL|ORDER_SUBMIT)\"" sentinel/
```

Reproduced locally: **exit 1 (no match) — clean.**

Timeouts: workflow does not declare a top-level `timeout-minutes`.
GitHub default is 6 hours, which is far more than needed for a
sweep that completes in under 60s locally. **Recommendation
(optional, deferred):** add `timeout-minutes: 15` at job level
to fail-fast on a hung step. Not a v0.1 blocker.

**Verdict: hardened. One optional polish item logged.**

### Findings

**D-1 (minor — doc phrasing drift).**
Several documents phrase the public-API count as
`"41 + __version__"` or `"41 symbols + __version__"`. The actual
`__all__` contains **41 entries total**, of which one IS
`__version__`. The accurate phrasing is one of:

```
- "41 symbols (including __version__)"
- "40 symbols + __version__ (41 total)"
```

Affected lines:

```
docs/reviews/0007-v0.1-release-readiness.md
docs/reviews/0010-v0.1-release-closure.md (status block)
docs/releases/v0.1.0-mvb.md  (the section heading "(41 symbols
                              + __version__)" should read
                              "(41 symbols including __version__)")
docs/releases/v0.1.0-mvb.tag-message.txt
    "public API:    41 symbols + __version__"
```

The GitHub Release body has already been published with the
"41 symbols + __version__" phrasing. Per release task §6
("uyumsuzsa overwrite etme") the published release body is NOT
modified. Only the in-repo documents will be corrected in a
follow-up doc-only commit if Halit approves.

**D-2 (none — no other drift detected).**

The MVP output set enumerated in every doc that lists it:
`{WAIT, BLOCK, MONITOR, NEED_RECALL, NO_ACTION}` — five members,
matches `SystemOutput` exactly.

**Verdict: one minor doc-phrasing drift; no factual
inconsistency.**
